A Prototype-Based Language

In JavaScript, prototypes allow objects to inherit features from one another.This is why JavaScript is often called a prototype-based language.

The Prototype Chain

In order to provide inheritance, objects can have a prototype object. Think of this as a template object that it inherits it's methods and properties from. An object's prototype object may also have a prototype object which it inherits it's properties and methods from. This creates the prototype chain.

One important thing to note about object prototypes, however,is that properties and methods are defined on the prototype property of the Object's constructor function, not the object instances themselves. It's important to understand that the object's prototype is the property on each instance and the prototype property on the constructor function is the property on the constructor. Methods and properties are not copied from one object to another in the prototype chain. They are accessed by walking up the chain as described above.

The Prototype Property

Inherited properties and methods are defind on the prototype property (think of it like a subnnamespace). The prototype property's value is an object, which acts as a storage container for storing properties and methods that we want objects further down the prototype chain to inherit.

The Constructor Property

Constructor functions prototype property object contains a constructor property that points to the original constructor function. This means that the constructor property is available to object instances. We can use this property to determine the name of the constructor function as well as create new object instances without referencing the original constructor function.

Modifying Prototypes

We can modify the prototype property on our constructor function. In fact, it's a good idea to add our methods to the prototype property object of our constructor function instead of inside the constructor function itself. By doing so, we ensure that methods are always available to all object instances created using the constructor function and we avoid defining our methods every time an object instance is created, making our code more performant.

Although it's a good idea to define our methods on the prototype property of our constructor function, properties are not as flexible. As such, we want to stick to creating our properties inside the constructor function.
